## Summary

The `SetGoalDialog` component has been implemented at `frontend/src/components/goals/SetGoalDialog.tsx`.

### Component Behavior

**Props:**
- `open` / `onOpenChange` — standard dialog open/close control
- `familyId`, `yearMonth`, `categoryId`, `categoryName` — context for the goal being set
- `existingGoal` — optional; when provided, dialog is in "edit" mode

**Title logic:**
- No existing goal: "Set Goal — {categoryName}"
- Existing goal: "Edit Goal — {categoryName}"

**Form:**
- Amount input with `inputMode="decimal"` and `type="number"`, auto-focused on open
- Pre-populated with existing goal amount when editing
- Save button disabled until a valid positive amount is entered

**Mutations:**
- New goal: calls `updateGoalsBulk()` with the single category goal
- Existing goal: calls `updateGoal()` with `expected_version` for optimistic locking
- On success: dual cache invalidation (`['goals', familyId, yearMonth]` and `['budget-summary', familyId, yearMonth]`)
- Success toast: "Goal set" or "Goal updated" with category name
- Error toast: descriptive error message, dialog remains open

**Design Patterns Followed:**
- Matches `CreateCategoryDialog.tsx` structure (DialogRoot, DialogBackdrop, DialogContent, etc.)
- GoalForm inner component uses `key` prop for clean re-mount on each open
- No setState in useEffect (ESLint `react-hooks/set-state-in-effect` rule respected)
- All props typed with TypeScript interfaces
