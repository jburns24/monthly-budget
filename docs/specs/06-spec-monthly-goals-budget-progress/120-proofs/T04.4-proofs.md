# T04.4 Proof Summary — Goals Management UI (DashboardPage Integration)

## Task

Integrate goal management UI components into `frontend/src/pages/DashboardPage.tsx`.

## Changes Made

### `frontend/src/pages/DashboardPage.tsx`
- Added `getGoals` query using React Query with `['goals', familyId, yearMonth]` key
- Added `getCategories` query (enabled only for admins) with `['categories', familyId]` key
- Extended `CategoryCard` with `isAdmin`, `existingGoal`, `onGoalClick` props
- Wrapped `CategoryCard` in `React.memo()` to prevent unnecessary re-renders
- Added "Set Goal +" / "Edit Goal ✏️" buttons on each category card (admin only)
- Added "Manage All Goals" button below category list (admin only)
- Added `SetGoalDialog` wired to individual goal editing
- Added `BulkGoalsEditor` wired to "Manage All Goals" button
- Added `RolloverPrompt` displayed when: admin + no goals for current month + previous month has goals
- Set `staleTime: 30_000` on budget-summary query to reduce unnecessary refetches

### `frontend/src/__tests__/DashboardPage.test.tsx`
- Added mocks for `../api/goals`, `../api/categories`, goal dialog components, `RolloverPrompt`
- Added `makeAdminWithFamily()` helper
- Added 8 new tests covering:
  - Admin sees "Set Goal" buttons when no goals exist
  - Admin sees "Edit Goal" button when a goal exists for a category
  - Member does NOT see goal buttons
  - Admin sees "Manage All Goals" button
  - Member does NOT see "Manage All Goals" button
  - Admin sees rollover prompt when no goals but previous month has goals
  - Member does NOT see rollover prompt
  - Rollover prompt hidden when current month already has goals
