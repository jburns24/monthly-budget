### Created `e2e/tests/monthly-goals.spec.ts`

New E2E test file with 6 test scenarios covering the full goal management workflow:

1. **Admin sets a goal via SetGoalDialog and progress bar appears**
   - Seeds expense via API so category card renders
   - Clicks `set-goal-btn-{categoryId}` to open SetGoalDialog
   - Fills `goal-amount-input` with $600, clicks `goal-save-btn`
   - Waits for PUT /goals (HTTP 200)
   - Verifies progress bar aria-label appears and `edit-goal-btn` replaces `set-goal-btn`

2. **Admin edits an existing goal and sees updated progress bar**
   - Seeds existing goal via `createMonthlyGoalViaApi`
   - Opens dialog via `edit-goal-btn-{categoryId}`
   - Updates amount to $1000, saves
   - Verifies updated amount in category card

3. **Admin uses BulkGoalsEditor to set goals for multiple categories**
   - Opens `manage-goals-btn`
   - Fills `goal-input-{categoryId}` for Groceries ($600) and Transport ($150)
   - Clicks `bulk-goals-save-btn`
   - Verifies both categories show their goals

4. **Rollover prompt appears and admin copies goals from previous month**
   - Seeds previous-month goals via `createMonthlyGoalViaApi`
   - Verifies `rollover-prompt` banner appears
   - Clicks `rollover-copy-btn`, waits for POST /goals/rollover (HTTP 200)
   - Verifies prompt disappears and goals are copied

5. **Admin dismisses rollover prompt with Start Fresh**
   - Clicks `rollover-start-fresh-btn`
   - Verifies prompt dismisses without API call

6. **Member sees progress bars but not goal management buttons**
   - Creates member (User B) who accepts family invite
   - Switches browser session to member
   - Verifies progress bar visible, but `edit-goal-btn`, `set-goal-btn`, `manage-goals-btn` all absent
