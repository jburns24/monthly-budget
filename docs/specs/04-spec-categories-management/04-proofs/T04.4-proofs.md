## Implementation Summary

Created `ArchiveCategoryDialog.tsx` following the Chakra UI v3 DialogRoot pattern established
in `RemoveMemberDialog.tsx` and `RoleChangeDialog.tsx`.

Key behaviors:
- Props: `open`, `onOpenChange`, `familyId`, `category: Category | null`
- Calls `deleteCategory(familyId, category.id)` via `useMutation`
- On success with `data.deleted === true`: toasts "Category deleted successfully"
- On success with `data.archived === true`: toasts "Category archived (N expense[s] reference it)"
  with correct singular/plural for expense count
- Invalidates `['categories', familyId]` query on success
- Uses `toaster.create()` for all user feedback
- Error handler shows generic failure toast
- Cancel button disabled while mutation is pending
- Delete button disabled when no category is selected
