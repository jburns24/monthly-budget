## Implementation Notes

### delete_category
- Queries by `category_id` AND `family_id` (scoped to family).
- Delegates expense check to `_count_category_expenses()` stub that returns 0 until the expenses table exists. Docstring contains the replacement query ready to paste in.
- Hard-deletes via `await db.delete(category)` + `await db.flush()` when no expenses.
- Archives (`is_active = False`) + `await db.flush()` when expenses exist.
- Returns `{"deleted": True}` or `{"deleted": False, "archived": True, "expense_count": N}`.
- Raises `HTTPException(404)` if not found.

### seed_default_categories
- Loads existing category names for family in one query (idempotency).
- Adds only missing categories from the 6 defaults (Groceries, Dining, Transport, Entertainment, Bills, Other) with sort_order 0-5 and emoji icons.
- Calls `await db.flush()` only when at least one category was created.
- Returns count of newly created rows.
