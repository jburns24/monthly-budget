## Implementation

Created `backend/app/services/category_service.py` following the `family_service.py` pattern.

### Functions Implemented

1. **create_category** - Creates a category, catches `IntegrityError` from the `uq_categories_family_name` unique constraint and raises `HTTPException(409)`. Uses `db.flush()` and structlog logging.

2. **list_active_categories** - Returns categories where `is_active=True`, ordered by `sort_order ASC, name ASC`.

3. **update_category** - Patches only non-None fields. Raises `HTTPException(404)` if category not found or belongs to wrong family. Raises `HTTPException(409)` on name conflicts.

### Pattern Compliance

- Uses `from app.logging import get_logger` (same as family_service.py)
- Uses `await db.flush()` (not commit)
- Uses `structlog` for all logging
- Uses async SQLAlchemy 2.0 patterns with `select()` + `scalar_one_or_none()`
- Handles `IntegrityError` for duplicate name detection
