## Test Scenarios Implemented

1. **Create via FAB on Dashboard** — Opens CreateExpenseDialog via FAB, submits expense, verifies it appears in ExpensesPage list and dashboard total updates.

2. **Create via Expenses Page** — Opens CreateExpenseDialog via header button, submits expense, verifies it appears in list, and category filter shows/hides it correctly.

3. **Edit Expense** — Seeds expense via API, clicks edit button, changes amount and description, submits via PUT, verifies updated values replace old ones.

4. **Delete Expense** — Seeds expense via API, clicks delete button, confirms via dialog, verifies expense removed from list and dashboard shows $0 total.

5. **Month Selector Filtering** — Seeds expenses in current and previous month, verifies dashboard month navigation updates totals, and expenses page month navigation shows correct filtered list.

6. **Dashboard Progress Colors** — Creates monthly goal ($100) and expense ($90 = 90%), verifies dashboard category card progress bar shows 90% aria-label (yellow/warning status).

7. **Multi-member Visibility** — Alice creates expense, Bob joins via invite, Bob creates his own expense, verifies both expenses visible in shared list with correct creator attribution.
