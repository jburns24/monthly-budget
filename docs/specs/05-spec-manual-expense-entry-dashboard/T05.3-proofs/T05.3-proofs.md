## Test Coverage

17 tests covering:

1. **Loading state** - spinner shown with `aria-label="Loading expenses"` while query pending
2. **Expense list renders** - cards with correct data displayed on load
3. **Month display** - month shown in header with `data-testid="month-display"`
4. **Month navigation** - prev/next month buttons present and change the month display
5. **Category filter shows** - select rendered when categories available
6. **Category filter hidden** - no filter when no categories
7. **Category filter change** - selecting a category triggers new API call with category_id
8. **Pagination shown** - controls visible when total_count > per_page (25 > 20)
9. **Pagination hidden** - no controls when total_count <= per_page (2 <= 20)
10. **Empty state** - shows `expense-list-empty` testid when no expenses
11. **Add Expense button shown** - button present when user has a family
12. **Add Expense button hidden** - button absent when user has no family
13. **No family message** - "create or join a family" text shown without family
14. **Dialog opens** - clicking Add Expense opens dialog
15. **Error state** - "Failed to load expenses" message shown on API error
16. **URL param initialization** - month initializes from `?month=` search param
17. **Month label with count** - shows expense count in summary line
