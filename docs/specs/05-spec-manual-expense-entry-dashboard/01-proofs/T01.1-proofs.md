### Files Modified
- `backend/app/models/__init__.py` — Added Expense and MonthlyGoal imports and exports
- `backend/app/models/family.py` — Added `expenses` and `monthly_goals` relationships with cascade + passive_deletes
- `backend/app/models/category.py` — Added `expenses` and `monthly_goals` relationships with passive_deletes
- `backend/app/models/user.py` — Added `expenses` relationship with passive_deletes
- `backend/app/routers/dev_auth.py` — Added expenses/monthly_goals to test reset truncation order

### Key Decisions
- Used `passive_deletes=True` on all new relationships to prevent SQLAlchemy from lazy-loading the non-existent `expenses`/`monthly_goals` tables during delete operations (tables are created in T01.2 migration)
- `category_id` FK uses `RESTRICT` (not CASCADE) for both expense and monthly_goal FK to categories
- `year_month` VARCHAR(7) stores ISO format like "2026-04"
