## Implementation Notes

### ExpensesPage POM (`e2e/pages/expenses.page.ts`)
Follows the same pattern as `categories.page.ts`. All locators target `data-testid`
attributes already present in `ExpensesPage.tsx`, `ExpenseList.tsx`, and
`CreateExpenseDialog.tsx`:
- Add expense button (`data-testid="add-expense-btn"`)
- FAB button (`data-testid="fab-add-expense"`)
- Month selector prev/next/display
- Category filter select
- Expense form inputs (amount, description, category, date, submit)
- Expense list and individual card locators
- Pagination controls

### DashboardPage POM (`e2e/pages/dashboard.page.ts`)
Uses accessible role and aria-label selectors since `DashboardPage.tsx` does not
include `data-testid` attributes on dashboard-specific elements (month heading,
total spent box, category cards). Category cards are located via
`[role="button"][aria-label$=" category"]` matching the `aria-label` in
`CategoryCard` component. FAB and expense form inputs use `data-testid` as usual.

### createExpenseViaApi helper
Matches the existing `createCategoryViaApi` pattern in `test-data.ts`. Posts to
`POST /api/families/{familyId}/expenses` with the `ExpenseCreate` schema fields.
TypeScript compiles without errors (`npx tsc --noEmit` exit code 0).
