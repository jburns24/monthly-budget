## Implementation Summary

Created `backend/app/services/expense_service.py` with the following functions:

### `_validate_category(db, family_id, category_id) -> Category`
Internal helper. Fetches category from DB, checks it exists, belongs to family, and `is_active=True`. Raises `HTTPException(400)` if invalid.

### `create_expense(db, family_id, user_id, category_id, amount_cents, description, expense_date) -> Expense`
- Validates category via `_validate_category` (raises 400 if invalid)
- Computes `year_month = expense_date.strftime("%Y-%m")`
- Creates Expense ORM object with `created_at` and `updated_at` set to `now()`
- Flushes, then reloads with `selectinload(Expense.category)` and `selectinload(Expense.user)`
- Logs `expense_created`

### `list_expenses(db, family_id, year_month, category_id=None, page=1, per_page=50) -> tuple[list[Expense], int]`
- Filters by `family_id` + `year_month`, optionally by `category_id`
- Runs count query for `total_count`
- Runs data query with `selectinload`, ordered `expense_date DESC, created_at DESC`, offset pagination
- Logs `expenses_listed`

### `get_expense(db, family_id, expense_id) -> Expense`
- Single query with `selectinload` for both relationships
- Raises `HTTPException(404)` if not found or wrong family

### `update_expense(db, family_id, expense_id, expected_updated_at, **fields) -> Expense`
- Fetches expense, raises 404 if not found
- Optimistic locking: compares `expense.updated_at` with `expected_updated_at` (normalizes tz), raises `HTTPException(409)` on mismatch
- Applies non-None fields from `**fields` via `setattr`
- Re-validates category if `category_id` in fields
- Re-computes `year_month` if `expense_date` in fields
- Sets `updated_at = now()`, flushes, reloads with selectinload
- Logs `expense_updated`

### `delete_expense(db, family_id, expense_id) -> None`
- Fetches expense, raises 404 if not found or wrong family
- Hard deletes via `db.delete()`, flushes
- Logs `expense_deleted`

## Patterns Followed

- Mirrors `category_service.py`: `flush()` not `commit()`, `HTTPException` for errors, `structlog` logger
- Uses `selectinload` (not lazy loading) for async-safe relationship loading
- All functions are `async`, take `AsyncSession` as first param after `db`
