## Implementation

Created `frontend/src/components/expenses/CreateExpenseDialog.tsx` with:

- **Props**: `open`, `onOpenChange`, `familyId`
- **Pattern**: Follows CreateCategoryDialog Chakra UI v3 DialogRoot pattern exactly
- **Form fields**:
  - Amount input: `inputMode="decimal"`, auto-focused via ref on open
  - Description input: free text, `maxLength={500}`
  - Category dropdown: `NativeSelectRoot/NativeSelectField` populated from `useQuery(['categories', familyId])`
  - Date input: `type="date"`, defaults to today via `todayString()`
- **Default category**: Derived from `localStorage.getItem('lastUsedCategoryId')` falling back to first category; avoids setState-in-effect ESLint rule by computing with `getDefaultCategoryId()` helper
- **Submission**: `useMutation` calling `createExpense(familyId, {...})`; converts decimal string to cents via `Math.round(parseFloat(amount) * 100)`
- **onSuccess**: Stores category ID in `localStorage`, invalidates `['expenses', familyId]` and `['budget-summary', familyId]` queries, shows success toast
- **onError**: Shows error toast
