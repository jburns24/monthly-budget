### API Client (frontend/src/api/expenses.ts)

All required functions implemented following the `categories.ts` pattern:
- `getExpenses(familyId, yearMonth, categoryId?, page?, perPage?)` — paginated list with filters
- `createExpense(familyId, data)` — POST with JSON body
- `getExpense(familyId, expenseId)` — single expense fetch
- `updateExpense(familyId, expenseId, data)` — PATCH with special 409 conflict handling
- `deleteExpense(familyId, expenseId)` — DELETE (returns void)
- `getBudgetSummary(familyId, month)` — GET budget summary for month

### Conventions Followed
- Uses `apiClient()` from `./client` for all requests
- Error handling with `if (!response.ok) throw new Error(...)` pattern
- Content-Type header set for POST/PATCH requests
- URLSearchParams for query string building in getExpenses
- Special 409 Conflict detection in updateExpense (throws `Error('CONFLICT')` for dialog-level handling)
