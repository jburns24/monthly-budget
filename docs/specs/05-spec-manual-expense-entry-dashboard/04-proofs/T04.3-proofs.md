## Implementation Notes

### EditExpenseDialog
- Location: `frontend/src/components/expenses/EditExpenseDialog.tsx`
- Follows `EditCategoryDialog` pattern with an inner `EditForm` sub-component
- Pre-populates all fields from the existing expense object
- Uses `key={expense.id + expense.updated_at}` to force form reset when expense changes
- Fetches categories with `useQuery(['categories', familyId])` for the dropdown
- Sends `expected_updated_at` field in the PATCH request for optimistic locking
- On 409 (CONFLICT error from `updateExpense`), shows toast: "This expense was modified by someone else. Please refresh and try again."
- Invalidates `['expenses', familyId]` and `['budget-summary', familyId]` on success

### DeleteExpenseDialog
- Location: `frontend/src/components/expenses/DeleteExpenseDialog.tsx`
- Follows `ArchiveCategoryDialog` pattern exactly
- Shows confirmation text with expense description and formatted amount (e.g., `$45.23`)
- Calls `deleteExpense(familyId, expense.id)` on confirm
- Invalidates `['expenses', familyId]` and `['budget-summary', familyId]` on success
