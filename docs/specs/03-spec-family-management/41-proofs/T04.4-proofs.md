# T04.4 Proof Summary: Role management, member removal, leave family, and pending invites UI

## Task
Build role management, member removal, leave family, and pending invites UI components.

## Components Created
1. **RoleChangeDialog.tsx** - Confirmation dialog to promote/demote members. Uses Chakra UI v3 Dialog, React Query useMutation with cache invalidation on ['family', familyId].
2. **RemoveMemberDialog.tsx** - Confirmation dialog to remove a member from the family. Red "Remove" button, invalidates family and currentUser queries.
3. **PendingInvites.tsx** - Lists pending invites for the current user with Accept/Decline buttons. Filters for pending status, invalidates pendingInvites + currentUser + family queries on response.
4. **LeaveButton.tsx** - "Leave Family" button with confirmation dialog. Hidden for family owner. Invalidates currentUser + family queries on success.

## Components Modified
5. **MemberList.tsx** - Added Promote/Demote and Remove buttons per member row (visible only to admins, not for self or owner). Integrated RoleChangeDialog and RemoveMemberDialog.
6. **FamilyDashboardView.tsx** - Added PendingInvites section and LeaveButton (hidden for owner). Passes ownerId and isAdmin to MemberList.
7. **HomePage.tsx** - Added PendingInvites section for authenticated users.
