## What was implemented

**`backend/tests/conftest.py`** was extended with:

- **`create_test_user(db, **overrides)`** — async factory function that inserts a `User` into the test DB with sensible defaults; returns the ORM object after flush+refresh.

- **`db_session`** — async fixture providing an `AsyncSession` bound to a per-test transaction that is always rolled back at teardown (no test data leaks between tests).

- **`authenticated_client`** — fixture factory returning a callable that accepts a `User` and returns an `httpx.AsyncClient` pre-loaded with `access_token` (15-min, HS256) and `refresh_token` (7-day, HS256) cookies.

- **`mock_google_oauth`** — fixture factory returning a callable that accepts a `user_info` dict and returns a `unittest.mock.patch` context manager patching `google.oauth2.id_token.verify_oauth2_token`.

## Key design decisions

- `_TEST_JWT_SECRET` falls back to a 40-char test-only secret when `settings.jwt_secret` is shorter than 32 chars (dev environment default), avoiding startup failures.
- Fixtures are composable: `db_session` + `create_test_user` + `authenticated_client` compose cleanly in integration tests.
- `mock_google_oauth` patches at the canonical path (`google.oauth2.id_token.verify_oauth2_token`) so future auth service code using the library will be intercepted regardless of import style.
- Both `RefreshTokenBlacklist` and `User` models are imported in `conftest.py` to ensure they register with `Base.metadata` before any `create_all` call.
