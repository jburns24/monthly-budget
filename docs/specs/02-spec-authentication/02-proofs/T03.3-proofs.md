## Key Implementation Details

- **LoginPage**: Chakra UI centered card layout; on button click: generateCodeVerifier + generateCodeChallenge (SHA-256/S256) + generateState; stores `pkce_code_verifier` and `oauth_state` in sessionStorage; redirects to Google OAuth endpoint with `code_challenge_method=S256`, `scope=openid email profile`; error displayed inline (no separate error page); reads `VITE_GOOGLE_CLIENT_ID` env var
- **AuthCallbackPage**: All sync validation (errorParam, missing code/state, state mismatch, missing verifier) derived at render time (avoids `react-hooks/set-state-in-effect` lint rule); useRef guard prevents React 18 Strict Mode double-execution; on success redirects to `/` or `/?new=true` depending on `is_new_user`
- **Routing**: App.tsx extended with `/login` and `/auth/callback` routes
- **Test coverage**: Login renders heading + button; callback shows correct error for each invalid state; loading spinner shown for valid in-flight callback
